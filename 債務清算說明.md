# 債務清算優化說明

## 📊 計算邏輯

### 第一步：計算每個人的淨餘額
```
淨餘額 = 已付金額 - 應付金額
```
- **正數**：表示這個人多付了（債權人）
- **負數**：表示這個人少付了（債務人）

### 第二步：根據情況選擇策略

#### 情況 A：正常結算（有人多付、有人少付）
使用「貪心算法」找出最優轉帳方案：
1. 將所有債權人按照多付金額從大到小排序
2. 將所有債務人按照少付金額從大到小排序
3. 配對最大債權人和最大債務人，計算轉帳金額
4. 重複直到所有人結清

#### 情況 B：所有人都多付 ⭐ 公平分攤模式
使用「平均分攤算法」達到完全公平：
1. 計算總多付金額
2. 平均分配：**平均值 = 總多付金額 ÷ 成員人數**
3. 計算每個人需要調整的金額：
   - 多付超過平均 → 應該拿回差額
   - 多付低於平均 → 應該補足差額
4. 配對「應拿回」和「應補足」的人，生成轉帳方案
5. 結算後，每個人都多付相同金額（平均值）

### 範例 2：所有人都多付 ⭐ 公平分攤模式

假設結算結果：
- 二哥：應付 $2,000，實付 $8,000 → **多付 $6,000**
- 大哥：應付 $2,000，實付 $5,000 → **多付 $3,000**
- 三弟：應付 $2,000，實付 $4,000 → **多付 $2,000**
- 倩瑜：應付 $2,000，實付 $3,000 → **多付 $1,000**

#### 步驟 1：計算平均值
```
總多付金額 = $6,000 + $3,000 + $2,000 + $1,000 = $12,000
平均每人應多付 = $12,000 ÷ 4 = $3,000
```

#### 步驟 2：計算每個人的調整金額
```
二哥：$6,000 - $3,000 = +$3,000（應拿回）
大哥：$3,000 - $3,000 = $0（剛好平衡）
三弟：$2,000 - $3,000 = -$1,000（應補足）
倩瑜：$1,000 - $3,000 = -$2,000（應補足）
```

#### 步驟 3：生成轉帳方案
✅ **只需 2 筆轉帳**：
1. 倩瑜 → 二哥：$2,000
2. 三弟 → 二哥：$1,000

#### 步驟 4：結算後的狀態
```
二哥：原本多付 $6,000，收到 $3,000 → 最終多付 $3,000 ✅
大哥：原本多付 $3,000，不變 → 最終多付 $3,000 ✅
三弟：原本多付 $2,000，轉出 $1,000 → 最終多付 $3,000 ✅
倩瑜：原本多付 $1,000，轉出 $2,000 → 最終多付 $3,000 ✅
```

**🎯 達成目標**：每個人都多付相同金額（$3,000），完全公平！

## 🎯 優點

### 1. **減少轉帳次數**
- 不需要所有錢都經過同一個人（如二哥）
- 直接讓欠款人轉給多付的人

### 2. **降低手續費**
- 轉帳次數越少，手續費越低

### 3. **避免現金流過度集中**
- 不會讓某個人（如二哥）短時間內大進大出
- 降低資金壓力

### 4. **完全公平（公平分攤模式）** ⭐ 新！
- 當所有人都多付時，自動平均分攤
- 讓每個人最終多付相同金額
- 不會有人承擔過多或過少

## ⚠️ 注意事項

### Q：如果所有人都多付，怎麼辦？
A：系統會自動啟動**公平分攤模式**：
1. 計算平均每人應該多付多少
2. 讓多付超過平均的人拿回差額
3. 讓多付低於平均的人補足差額
4. 最終每個人都多付相同金額，達到完全公平

### Q：為什麼要讓大家都多付相同金額？
A：這是最公平的方式！舉例：
- 如果二哥多付 $6,000，其他人多付 $1,000
- 不平衡感會很強，對二哥不公平
- 透過相互抵銷，讓大家承擔相同的「多付」金額
- 這筆錢可以成為公用基金，下期使用

### Q：上期餘額如何影響？
A：
- 上期多付的人，本期應付金額會減少
- 上期少付的人，本期應付金額會增加
- 可以在應用中關閉「套用上期餘額」功能

### Q：小數點怎麼處理？
A：系統會自動四捨五入到整數，忽略小於 1 元的誤差。

## 🔧 演算法說明（技術細節）

### 公平分攤算法（所有人都多付時）

```javascript
// 公平分攤算法
function fairSplitAllOverpaid(balances) {
    // 1. 計算平均值
    const totalOverpaid = balances.reduce((sum, b) => sum + b.net, 0);
    const average = totalOverpaid / balances.length;
    
    // 2. 計算每個人相對於平均值的調整
    const adjustments = balances.map(b => ({
        member: b.member,
        original: b.net,
        adjustment: b.net - average  // 正數=退還，負數=補足
    }));
    
    // 3. 配對需要退款和需要補款的人
    const needRefund = adjustments.filter(a => a.adjustment > 0);
    const needPay = adjustments.filter(a => a.adjustment < 0);
    
    // 4. 生成轉帳方案（貪心配對）
    const transfers = [];
    let i = 0, j = 0;
    
    while (i < needRefund.length && j < needPay.length) {
        const amount = Math.min(needRefund[i].adjustment, 
                               Math.abs(needPay[j].adjustment));
        
        transfers.push({
            from: needPay[j].member,
            to: needRefund[i].member,
            amount: amount
        });
        
        needRefund[i].adjustment -= amount;
        needPay[j].adjustment += amount;
        
        if (needRefund[i].adjustment === 0) i++;
        if (needPay[j].adjustment === 0) j++;
    }
    
    return { transfers, average };
}
```
